# Картотека — архив нераскрытых дел

### ▶ [Начать расследование](https://detective-case-001.vercel.app/)

Интерактивные детективные дела, которые посетитель расследует сам. Не читает чужое
расследование — ведёт своё: рассматривает улики, ловит подозреваемых на лжи,
восстанавливает ход ночи и называет виновного.

> **Здесь не угадывают. Здесь исключают.**

Регистрация не нужна: зашёл и играешь, прогресс хранится в браузере.

## Дела в архиве

| № | Дело | Сложность | Ловушка |
|---|------|-----------|---------|
| 001 | **«Час, которого не было»** | Среднее | Двое часов показывают разное время. Один из них лжёт |
| 002 | **«Гроза по расписанию»** | Сложное | В театре все на виду. Значит, алиби есть у всех — и это первая ложь |

Первое дело открыто всегда, следующее — после раскрытия предыдущего (и есть кнопка
«открыть все» для тех, кто не хочет идти по порядку).

## Как ведётся расследование

| Экран | Механика |
|---|---|
| **Доска улик** | Пробковая доска: карточки-улики разных носителей (полароид, бланк, записка на скотче, чек, порванный документ). Клик открывает улику вблизи; оттуда тянется SVG-нить к подозреваемому — карта версий игрока |
| **Подозреваемые** | Досье с мотивом и алиби; портреты — рисованные силуэты в стиле полицейского фото |
| **Допрос** | Ветвящийся диалог. Каждый ответ сверяется с уликой: алиби либо подтверждается, либо **противоречит** — противоречие подсвечивается |
| **Хронология** | Drag & drop. Время скрыто — порядок восстанавливается логикой. Ошибка даёт подсказку «не сходится по времени» |
| **Обвинение** | Подозреваемый + мотив + улика-доказательство |
| **Разгадка** | Полный разбор — **только за верное обвинение**. Ошибся — ответа не будет, возвращайся к уликам |

### Дедукция — это механика, а не декорация

В каждом деле улики ведут к **одному** виновному через исключение: у остальных алиби
подтверждается твёрдым фактом. Ложные следы есть, но каждый опровергается другой уликой —
отпечатки подмастерья объясняются его ежедневной работой, а глухая маска на сцене не
скрывает голос на записи. Ни одна деталь не требует догадки.

## Стек

- **React 18 + Vite 6 + react-router** — состояние расследования общее для всех экранов,
  поэтому React оправдан; маршруты дают ссылки на дела (`/case/001`)
- **Дела — данные, а не код.** Новое дело = файл `src/cases/case-00X.js` + строка в реестре
  `src/cases/index.js`. Обложки грузятся сразу, само дело — лениво, отдельным чанком:
  десяток дел не утяжеляет первую загрузку
- **Доска** — фиксированный холст 1040×640, масштабируется под экран целиком. Геометрия
  нитей считается арифметикой по раскладке дела, без замеров DOM
- **Графика вся векторная** — портреты, иконки улик, стол детектива. Никаких внешних картинок
- **Прогресс** — `localStorage`, отдельно на каждое дело. Бэкенда нет
- Зависимостей пять: `react`, `react-dom`, `react-router-dom`, `vite`, `@vitejs/plugin-react`

### Визуальный стиль

Нуар: чёрно-белая база и **один** акцент — янтарный (свет лампы). Печатная машинка
(Special Elite / Courier Prime) для «документов», чистый sans-serif для интерфейса.
Зернистость плёнки, виньетирование, мерцание лампы, курсор-лупа. Акцент вынесен
в CSS-переменную `--accent` — меняется одной строкой. Учтён `prefers-reduced-motion`.

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # статика в dist/
npm run preview  # предпросмотр сборки
```

На выходе статический сайт для любого хостинга. Роутинг клиентский, поэтому нужен
SPA-редирект на `index.html` — настройки уже лежат в `vercel.json` и `netlify.toml`.

## Структура

```
├── content/                 # сюжетные библии дел (СПОЙЛЕРЫ)
│   ├── case-001.md
│   └── case-002.md
├── src/
│   ├── cases/
│   │   ├── index.js         # реестр: обложки + ленивая загрузка
│   │   ├── case-001.js      # модель дела: улики, допрос, хронология, разгадка, раскладка
│   │   └── case-002.js
│   ├── engine/screens.js    # экраны расследования — общее для всех дел
│   ├── state/
│   │   ├── progress.js      # прогресс по делам + логика замков
│   │   └── GameContext.jsx  # текущее дело + действия
│   ├── components/          # доска, портреты, иконки, стол детектива
│   ├── screens/             # главный, картотека, дело (7 экранов)
│   └── styles/global.css    # палитра, шрифты, зерно, лампа
└── CLAUDE.md                # исходное техзадание проекта
```

## ⚠️ Спойлеры

`content/*.md` и `src/cases/*.js` содержат **полные решения дел** — кто убийца, почему
и какая улика изобличает. Сайт статический, так что разгадка неизбежно уезжает в браузер:
это как задачник, где ответы в конце книги. Хотите расследовать сами — сначала пройдите
дела, потом читайте исходники.
