# Проект: Детективный сайт — "Дело №001"

## Концепция
Интерактивный сайт, на котором посетитель сам расследует вымышленное преступление,
используя логику и дедукцию — в духе Шерлока Холмса и сериала "Менталист"
(только приём мышления и атмосфера, БЕЗ использования брендов, имён персонажей
или конкретных сюжетов этих произведений — история полностью оригинальная).

Игрок не угадывает наугад: улики логически ведут к одному виновному через
исключение противоречий (алиби, показания, временная линия).

## Структура сайта / пользовательский путь

1. **Заставка** — папка/конверт с грифом открывается анимацией,
   появляется название дела: "Дело №001: [название]"
2. **Доска улик** — пробковая доска, на неё приколоты карточки: фото места
   преступления, записка, орудие, билет/чек-алиби, показания свидетеля.
   Каждая улика кликабельна, даёт кусочек правды
3. **Подозреваемые** — 3–4 карточки-досье с фото и алиби; одно алиби
   противоречит уликам
4. **Допрос** — диалог с ветвлением: игрок выбирает вопросы, ответы либо
   совпадают с уликами, либо противоречат им (противоречие подсвечивается)
5. **Хронология** — нужно расставить события в правильном порядке
   (drag & drop); неверный порядок даёт подсказку "не сходится по времени"
6. **Обвинение** — игрок выбирает подозреваемого + мотив + улику-доказательство
7. **Разгадка** — независимо от результата, полный разбор дела: кто виноват,
   почему, на какую деталь стоило обратить внимание. Это ключевой момент —
   объяснение логики, а не просто "верно/неверно"

## Визуальный стиль
- Атмосфера: нуар / доска расследования
- Палитра: чёрно-белая база + один акцент — красный (нити, печати) ИЛИ
  янтарный (свет лампы). Не смешивать оба
- Шрифты: печатная машинка (Special Elite, Courier Prime) для "документов",
  чистый sans-serif для интерфейса
- Текстуры: лёгкая зернистость плёнки, виньетирование, мерцание лампы
- Курсор: лупа на интерактивных элементах
- Анимации: карточки "прикалываются"/"открепляются", нити прорисовываются
  между уликами и подозреваемыми, папки открываются как настоящие файлы

## Механика дедукции (ключевая часть, не украшение)
- Улики должны логически указывать на одного подозреваемого через исключение
- Красные селёдки (ложные следы) допустимы, но должны быть опровержимы другой уликой
- Не делать финальный выбор случайным угадыванием — каждая деталь должна
  быть "честной" зацепкой

## Технологический стек (предложение, обсудить перед реализацией)
- Фронтенд: React + Vite ИЛИ чистый HTML/CSS/JS (если не нужен сложный стейт)
- Доска и нити: Canvas или SVG (динамические линии между узлами)
- Анимации: GSAP или CSS-переходы
- Стейт (собранные улики, ответы допроса, порядок хронологии, финальный
  выбор) — на клиенте, можно с localStorage для сохранения прогресса
- Бэкенд не нужен — статический сайт

## Контент — предстоит написать
- [ ] Сама история дела: жертва, преступление, 3–4 подозреваемых, мотивы,
      хронология событий, улики, ложные следы
- [ ] Диалоговые ветки допроса для каждого подозреваемого
- [ ] Финальный текст разгадки с разбором логики
- [ ] Ассеты: фото-заглушки/иллюстрации улик и подозреваемых

## Важно
- Никаких прямых отсылок к "Шерлоку Холмсу" или "Менталисту" (имена,
  внешний вид, цитаты, сюжеты) — только жанровая атмосфера и приём дедукции
- Проект для личного портфолио — код и история должны быть полностью
  оригинальными
